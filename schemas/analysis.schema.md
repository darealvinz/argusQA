# Spec Analysis Report Schema

Analysis reports are generated by the `spec-analyzer` skill after reading all input documents for a feature. The report is produced BEFORE the feature file — it proves the AI understood the specs correctly.

## File Naming

`<feature-id>.analysis.md`

**Location:** `.argus/reports/analysis/`

## Frontmatter

```yaml
---
id: <feature-id>-analysis
feature: <feature-id>
timestamp: <ISO 8601>
sources:
  - file: <filename>
    type: <business | frontend | backend | other>
total_requirements: <number>
total_gaps: <number>
total_questions: <number>
status: <ready | needs-input>
---
```

Status:
- **ready** — No gaps or questions, can proceed to feature file
- **needs-input** — Gaps or questions require user input before proceeding

## Sections

### 1. Sources Analyzed
Table with columns:
| File | Type | Content Found |

Type: business, frontend, backend, other
Content Found: brief summary (e.g., "8 requirements, 3 validation rules")

### 2. Requirements Found

Grouped by source type:

#### From Business
Numbered list of requirements extracted from business documents (SRS, PRD, user stories).
Each tagged with source file: `[source: srs.docx]`

#### From Frontend
Numbered list of UI elements, states, error messages, and layout requirements.
Each tagged with source file: `[source: wireframe.png]`

#### From Backend
Numbered list of API endpoints, status codes, data schemas, validation rules.
Each tagged with source file: `[source: api-contract.yaml]`

### 3. UI Elements Identified
Table with columns:
| Element | Source | Type | Notes |

Extracted from wireframes, designs, and UI spec documents via AI vision or text extraction.

### 4. API Endpoints Mapped
Table with columns:
| Method | Endpoint | Request | Response | Status Codes | Source |

Extracted from API contracts, backend specs.

### 5. Gap Analysis
Table with columns:
| Gap | Found In | Missing From | Action Needed |

Gap types:
- In Frontend but not Backend (UI exists, no API support)
- In Backend but not Frontend (API exists, no UI for it)
- In Business but not Frontend/Backend (requirement stated but not designed/built)
- In Frontend/Backend but not Business (implemented but not in requirements)

### 6. Questions for Tester
Numbered list of ambiguities, missing info, and decisions needed.
Each question should reference the specific gap or requirement it relates to.

### 7. Summary
- Total requirements extracted (by source type)
- Total gaps found
- Total questions needing answers
- Recommendation: "Ready to generate feature file" or "X questions need answers first"
