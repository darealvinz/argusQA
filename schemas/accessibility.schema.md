# Accessibility Report Schema

Accessibility reports are generated by the `accessibility-checker` skill when checking WCAG compliance.

## File Naming

`<feature-id>.accessibility.md`

**Location:** `.argus/reports/accessibility/`

## Frontmatter

```yaml
---
id: <feature-id>-accessibility
feature: <feature-id>
url: <page URL or "screenshot-only">
timestamp: <ISO 8601>
wcag_level: <A | AA | AAA>
input_type: <url | screenshot>
total_issues: <number>
critical_issues: <number>
status: <pass | fail>
---
```

## Sections

### 1. Summary
WCAG level tested, input type (URL or screenshot), total issues found, critical count, pass/fail status.

### 2. Automated Checks (axe-core)
Only present when input is a URL (axe-core requires DOM access).

Table with columns:
| # | Rule | Impact | Element | Description | WCAG Criteria |
Impact: critical | serious | moderate | minor

### 3. Visual Checks (AI Vision)
Checks that axe-core cannot perform, assessed via AI vision analysis.

Table with columns:
| # | Check | Status | Details |
Checks include:
- Focus indicators visible
- Logical reading order
- Touch target sizing (≥44x44px for AA)
- Text over images readability
- Motion/animation concerns

### 4. Color Contrast
Table with columns:
| Element | Foreground | Background | Ratio | Required | Status |
Required ratio: 4.5:1 for normal text, 3:1 for large text (WCAG AA).
Values from computed CSS styles when URL provided; AI estimation when screenshot-only (flagged as approximate).

### 5. Heading Hierarchy
List of headings found in order:
- h1: "Page Title"
  - h2: "Section"
    - h3: "Subsection"
Flag any skipped levels (e.g., h1 → h3 with no h2).

### 6. Form Accessibility
Table with columns:
| Field | Label | ARIA | Required Indicator | Error Handling | Status |
Checks: associated label, aria-label/aria-labelledby, required indicator, error message association.

### 7. Keyboard Navigation
Table with columns:
| Action | Key | Expected Behavior | Status |
Common checks: Tab order, Enter/Space activation, Escape to close, Arrow keys in menus.

### 8. Dynamic Content Accessibility
Table with columns:
| Component | Type | ARIA Role | Focus Trap | Announce | Status |
Components: modals, dropdowns, toasts, tabs, accordions.
Tested via Playwright interaction when URL provided.

### 9. Recommendation
AI-generated summary with prioritized fix list.

## Screenshot-Only Limitations

When only a screenshot is provided (no URL), the report flags:
- axe-core checks SKIPPED (requires DOM)
- Color contrast values are APPROXIMATE (AI estimation, not computed)
- Keyboard navigation SKIPPED (requires interaction)
- Dynamic content SKIPPED (requires interaction)
- Form accessibility PARTIAL (visual checks only, no ARIA inspection)

## WCAG Level Configuration

Read from `.argus/config.yaml` under `accessibility`:

| Setting | Config Key | Default |
|---------|------------|---------|
| WCAG level | `accessibility.wcagLevel` | AA |
| Ignore rules | `accessibility.ignoreRules` | [] |
