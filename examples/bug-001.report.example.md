# BUG-001: No rate limiting on login endpoint after 5 failed attempts

**Severity:** Major
**Feature:** login
**TestcaseID:** TC-LOGIN-032
**Environment:** Chrome 120, Desktop 1920x1080, https://staging.myapp.com
**Found by:** test-runner

## Description
The login endpoint does not enforce account lockout or CAPTCHA after 5 consecutive failed login attempts. The spec requires protection after 5 failed attempts.

## Steps to Reproduce
1. Navigate to /login
2. Enter email: john.doe@example.com
3. Enter wrong password: "WrongPass1!"
4. Click "Sign In"
5. Repeat steps 2-4 four more times (5 total failed attempts)
6. Enter the correct password on the 6th attempt

## Expected Result
- After 5th failed attempt: account is locked OR CAPTCHA is displayed
- 6th attempt with correct password should be blocked until lockout expires

## Actual Result
- All 5 failed attempts accepted without any protection
- 6th attempt with correct password succeeded — no lockout triggered
- No CAPTCHA displayed at any point

## Evidence
- Console errors: None
- Network: All POST /api/auth/login returned 401, no 429 or lockout response
- No rate-limit headers observed in responses

## Additional Context
- Related test cases: TC-LOGIN-032, TC-LOGIN-035
- Related features: login
- Spec requirement: "Account locks after 5 failed attempts"
